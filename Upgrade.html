Upgrade Package

I. Purpose & Context I. Mục đích & bối cảnh dùng

Upgrade Package is a lightweight toolset I built to handle the progression/upgrade system in games: scaling damage, firing rates, magazine sizes, health pools, etc., incrementally across levels and custom paths. Upgrade Package là một bộ công cụ nhỏ mình làm để xử lý phần hệ thống nâng cấp (progression) trong game: tăng damage, tăng tốc độ bắn, mở rộng số lượng đạn, tăng HP, v.v... theo từng cấp (level) và từng nhánh (branch).

Key Objectives: Mục tiêu chính:

  • Decouple progression logic from core gameplay (shooting, moving, etc.). Tách logic nâng cấp khỏi gameplay core (bắn, di chuyển...).
  • Allow **data-driven upgrade design** (ScriptableObject/config) instead of writing new code for every addition. Cho phép thiết kế Upgrade bằng data (ScriptableObject / config) thay vì viết lại code mỗi lần.
  • Easy reuse across various 2D/3D projects. Dễ tái sử dụng cho nhiều dự án 2D/3D khác nhau.

I utilize this package in roguelike/survivor/tower defense games where numerous small upgrades need to be easily added or removed. Package này mình dùng trong các game dạng roguelike / survivor / tower defense – nơi cần rất nhiều nâng cấp nhỏ nhưng phải dễ thêm/bớt.

Demo Video: illustrates the upgrade selection UI and how the effects apply dynamically to the character. Video demo: minh họa UI chọn Upgrade và hiệu ứng khi áp dụng vào nhân vật (bạn tự gắn video thật vào đây).

II. Key Features II. Tính năng chính

  • Branch-based Supports multiple upgrade branches (Offense / Defense / Utility...).Hỗ trợ nhiều nhánh nâng cấp (Offense / Defense / Utility...).
  • Level-based Each branch has multiple levels, applying distinct attribute modifiers.Mỗi nhánh có nhiều cấp, mỗi cấp áp dụng một tập hiệu ứng.
  • Data-driven All upgrade metadata (name, description, icon, cost, modifiers...) is defined via data.Toàn bộ thông tin nâng cấp (tên, mô tả, icon, giá, hiệu ứng...) được định nghĩa bằng data.
  • Event-based On upgrade selection, the package fires events for core gameplay systems to consume.Khi chọn Upgrade, package bắn event để gameplay lắng nghe và xử lý.
  • UI-ready Exposes appropriate data structures to bind directly to upgrade selection UIs.Chuẩn bị sẵn cấu trúc data phù hợp để bind lên UI lựa chọn nâng cấp.

III. Integration Flow (High Level) III. Cách tích hợp (mô tả khái quát)

  1. Declare Upgrade Data: Create ScriptableObjects (e.g., UpgradeBranch, UpgradeLevel) to define paths and levels. Khai báo data Upgrade: Tạo các ScriptableObject (ví dụ: UpgradeBranch, UpgradeLevel) để mô tả từng nhánh và từng cấp.
  2. Attach to UpgradeManager: A central component (UpgradeManager) manages active branches, computes next-level requirements, and fires events upon acquisition. Gắn vào hệ thống UpgradeManager: Một component trung tâm (UpgradeManager) sẽ giữ danh sách các nhánh, xử lý logic chọn cấp tiếp theo và bắn event khi người chơi nhận Upgrade.
  3. Gameplay Listens to Events: Player, Weapon, or Spawners subscribe to upgrade events to apply modifications. Gameplay lắng nghe event: Player / Weapon / EnemySpawner... subscribe event Upgrade để áp dụng thay đổi (tăng damage, tăng tốc, spawn nhiều hơn...).
  4. UI Displays Options: The UI fetches available options from UpgradeManager, randomizes 2–3 options, displays details, and invokes ApplyUpgrade when clicked. UI hiển thị lựa chọn: UI lấy list Upgrade khả dụng từ UpgradeManager, random 2–3 option, show icon + mô tả, và gọi hàm ApplyUpgrade khi người chơi bấm chọn.

IV. Notes & Extensions IV. Ghi chú & hướng mở rộng

  • Built with easy extensibility in mind: to add new upgrades, simply define new data assets and handle the event in the appropriate system. Package được viết theo hướng mở rộng dễ: muốn thêm kiểu Upgrade mới chỉ cần tạo thêm data + xử lý trong nơi lắng nghe event.
  • Can be integrated alongside Object Pooling or Enemy Spawners to generate diverse gameplay synergies. Có thể gắn chung với Object Pooling hoặc Enemy Spawner để tạo các build gameplay rất khác nhau mà vẫn dùng chung core.
  • Future expansions: support upgrade synergies (combos) or prerequisite unlocks (e.g., wave number, stat thresholds). Dự định nâng cấp thêm: hỗ trợ synergy giữa nhiều Upgrade, hoặc Unlock bằng điều kiện (chơi đủ số wave, đạt đủ stat...).
