🔧 Design Patterns in Unity

⭐ Quick Introduction to Design Patterns ⭐ Giới thiệu nhanh về Design Pattern

1. Who created Design Patterns? 1. Ai là người làm ra Design Pattern?

Design patterns in programming were popularized by four authors, commonly known as the Gang of Four (GoF): Design pattern trong lập trình được phổ biến bởi bốn tác giả, thường gọi là Gang of Four (GoF):

  • Erich Gamma
  • Richard Helm
  • Ralph Johnson
  • John Vlissides

They published a book in 1994: 📘 Design Patterns: Elements of Reusable Object-Oriented Software. This book laid the foundation for most patterns used today. Họ xuất bản cuốn sách năm 1994: 📘 Design Patterns: Elements of Reusable Object-Oriented Software. Cuốn sách này đặt nền móng cho hầu hết các pattern được dùng ngày nay.

2. What is the purpose of Design Patterns? 2. Mục đích của Design Pattern là gì?

🎯 1. Solve recurring OOP architectural problems 🎯 1. Giải quyết các vấn đề kiến trúc OOP lặp lại

  • Create flexible objects, manage dependencies, organize behaviors, facilitate communication between objects, and extend without breaking old code. Tạo object linh hoạt, quản lý phụ thuộc, tổ chức hành vi, giao tiếp giữa đối tượng, mở rộng mà không phá code cũ.
  • Patterns provide proven solutions rather than reinventing the wheel. Pattern cung cấp giải pháp đã được kiểm chứng, thay vì tự nghĩ lại từ đầu.

🎯 2. Make code readable, maintainable, and extensible 🎯 2. Giúp code dễ hiểu, dễ bảo trì, dễ mở rộng

  • Reduce coupling, increase reusability. Giảm phụ thuộc (coupling), tăng tái sử dụng.
  • Keep object responsibilities clear, supporting the OCP (Open/Closed Principle). Giữ rõ ràng trách nhiệm của object, hỗ trợ OCP (Open/Closed Principle).

🎯 3. Establish a common language among developers 🎯 3. Tạo ngôn ngữ chung giữa lập trình viên

  • Example: "Use the Strategy pattern" → everyone understands the context and solution, no long explanations needed. Ví dụ: “Dùng Strategy pattern” → ai cũng hiểu vấn đề và giải pháp, không cần giải thích dài dòng.

🎯 4. Write professional and structured code 🎯 4. Viết code chuyên nghiệp và có cấu trúc

  • Design patterns represent the accumulated experience of generations of developers — using them is standing on the shoulders of giants. Design pattern là kinh nghiệm tích lũy của nhiều thế hệ lập trình viên — sử dụng chúng là đứng trên vai người khổng lồ.

📍 Singleton Pattern — Unity Implementation (Clean & Safe)

⚠️ Common Problems with Traditional Singletons ⚠️ Vấn đề thường gặp với Singleton truyền thống

  • Easily duplicates instances during fast scene loading Dễ bị duplicate instance khi load scene nhanh
  • Public static fields → vulnerable to being overwritten externally Static field public → dễ bị ghi đè từ bên ngoài
  • Often devolves into a God Object → violates SOLID principles Thường biến thành God Object → vi phạm SOLID

✅ My Current Solution (Studio-Standard) ✅ Giải pháp em đang dùng (studio-standard)

```csharp public class GameManager : MonoBehaviour { public static GameManager Instance { get; private set; } private void Awake() { if (Instance != null && Instance != this) { Destroy(gameObject); return; } Instance = this; DontDestroyOnLoad(gameObject); } } ```

🎯 Key Advantages 🎯 Ưu điểm

  • 100% thread-safe on the Unity main thread 100% thread-safe trong Unity main thread
  • Zero duplication (validated by reloading scenes 100 times consecutively) Không bao giờ bị duplicate (đã test load scene liên tục 100 lần)
  • Private set property → secure, preventing external overrides Property private set → an toàn, không bị ghi đè từ bên ngoài
  • Single responsibility → ensures unique existence + integrates perfectly with SOLID services Chỉ làm đúng 1 việc → đảm bảo tồn tại duy nhất + kết hợp hoàn hảo với SOLID services
  • Readable and maintainable → highly recommended by seniors & major studios Dễ đọc, dễ bảo trì → được các senior & studio lớn khuyến khích

💡 Currently, all Managers in my projects 💡 Hiện tại mọi Manager trong dự án của em

GameManager, AudioManager, SaveManager... all utilize this pattern. GameManager, AudioManager, SaveManager… đều sử dụng pattern này.
